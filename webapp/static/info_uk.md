# AISTATE Web — Інформація

**AISTATE Web** (*Artificial Intelligence Speech‑To‑Analysis‑Translation‑Engine*) — це веб-додаток для **транскрипції**, **діаризації мовців**, **перекладу**, **аналізу GSM/BTS** та **фінансового аналізу AML**.

---

## 🚀 Що він робить

- **Транскрипція** — Аудіо → текст (Whisper, WhisperX, NeMo)
- **Діаризація** — "хто говорив коли" + сегменти мовців (pyannote, NeMo)
- **Переклад** — Текст → інші мови (NLLB‑200, повністю офлайн)
- **Аналіз (LLM / Ollama)** — резюме, висновки, звіти
- **Аналіз GSM / BTS** — імпорт білінгу, карта BTS, маршрути, кластери, хронологія
- **Фінансовий аналіз (AML)** — розбір банківських виписок, оцінка ризиків, виявлення аномалій
- **Журнали та прогрес** — моніторинг завдань + діагностика

## 🆕 Що нового у 3.6 beta

### 📱 Аналіз GSM / BTS
- Імпорт білінгових даних (CSV, XLSX, PDF)
- Інтерактивна **карта BTS** з кількома режимами перегляду: точки, шлях, кластери, поїздки, покриття BTS, теплова карта, хронологія
- **Офлайн-карти** — підтримка MBTiles (растрові PNG/JPG/WebP + векторні PBF через MapLibre GL)
- **Шари накладання**: військові бази, цивільні аеропорти, дипломатичні представництва (вбудовані дані)
- **Імпорт KML/KMZ** — користувацькі шари з Google Earth та інших ГІС-інструментів
- Вибір області (коло / прямокутник) для просторових запитів
- Граф контактів, теплова карта активності, аналіз топ-контактів
- Знімки карти з водяним знаком (онлайн та офлайн-карти + всі шари накладання)

### 💰 Фінансовий аналіз (AML)
- Конвеєр **протидії відмиванню коштів** для банківських виписок
- Автоматичне визначення банку та розбір PDF: PKO BP, ING, mBank, Pekao SA, Santander, Millennium, Revolut (+ універсальний резервний варіант)
- Підтримка формату виписок MT940 (SWIFT)
- Нормалізація транзакцій, класифікація на основі правил та оцінка ризиків
- **Виявлення аномалій**: статистична базова лінія + ML (Isolation Forest)
- **Аналіз графів** — візуалізація мережі контрагентів
- Крос-рахунковий аналіз для розслідувань з кількома рахунками
- Аналіз витрат, поведінкові патерни, категоризація торговців
- Аналіз за допомогою LLM (конструктор промптів для моделей Ollama)
- HTML-звіти з діаграмами
- Профілі анонімізації даних для безпечного обміну

## 📦 Звідки завантажуються моделі

AISTATE web **не** містить ваг моделей у репозиторії. Моделі завантажуються за потребою та кешуються локально (залежно від модуля):

- **Hugging Face Hub**: pyannote + NLLB (стандартний кеш HF).
- **NVIDIA NGC / NeMo**: моделі ASR/діаризації NeMo (поведінка кешування NeMo/NGC).
- **Ollama**: LLM-моделі завантажуються сервісом Ollama.

## 🔐 Безпека та управління користувачами

AISTATE Web підтримує два режими розгортання:

- **Однокористувацький режим** — спрощений, без входу в систему (локальний / self-hosted).
- **Багатокористувацький режим** — повна автентифікація, авторизація та управління обліковими записами (розрахований на 50–100 одночасних користувачів).

### 👥 Ролі та дозволи

**Ролі користувачів** (доступ до модулів):
- Transkryptor, Lingwista, Analityk, Dialogista, Strateg, Mistrz Sesji

**Адміністративні ролі:**
- **Architekt Funkcji** — управління налаштуваннями додатку
- **Strażnik Dostępu** — управління обліковими записами користувачів (створення, затвердження, блокування, скидання паролів)
- **Główny Opiekun (superadmin)** — повний доступ до всіх модулів та адміністративних функцій

### 🔑 Механізми безпеки

- **Хешування паролів**: PBKDF2-HMAC-SHA256 (260 000 ітерацій)
- **Політика паролів**: налаштовувана (відсутня / базова / середня / сильна); адміністратори завжди вимагають сильних паролів (12+ символів, змішаний регістр, цифра, спецсимвол)
- **Чорний список паролів**: вбудований + користувацький список адміністратора
- **Термін дії пароля**: налаштовуваний (примусова зміна через X днів)
- **Блокування облікового запису**: після налаштовуваної кількості невдалих спроб (за замовчуванням 5), автоматичне розблокування через 15 хв
- **Обмеження частоти**: вхід та реєстрація обмежені (5 на хвилину на IP)
- **Сесії**: безпечні токени (модуль secrets), куки HTTPOnly + SameSite=Lax, налаштовуваний тайм-аут (за замовчуванням 8 год)
- **Фраза відновлення**: 12-слівна мнемоніка BIP-39 (~132 біти ентропії) для самостійного відновлення пароля
- **Блокування користувачів**: постійне або тимчасове, із зазначенням причини
- **Заголовки безпеки**: X-Content-Type-Options, X-Frame-Options, X-XSS-Protection, Referrer-Policy

### 📝 Аудит та журналювання

- Повний журнал подій: входи, невдалі спроби, зміни паролів, створення/видалення облікових записів, блокування, розблокування
- Запис IP-адреси та відбитка браузера
- Файлові журнали з погодинною ротацією + база даних SQLite
- Історія входів користувача + повний аудиторський слід для адміністраторів

### 📋 Реєстрація та затвердження

- Самостійна реєстрація з обов'язковим затвердженням адміністратором (роль Strażnik Dostępu)
- Обов'язкова зміна пароля при першому вході
- Фраза відновлення генерується та відображається одноразово
