# AISTATE Web — 사용자 매뉴얼

---

## 1. 프로젝트

프로젝트는 AISTATE 작업의 핵심 요소입니다. 각 프로젝트에는 오디오 파일, 전사 결과, 화자분리, 번역, 분석 및 메모가 저장됩니다.

### 프로젝트 만들기
1. 사이드바에서 **프로젝트** 탭으로 이동합니다.
2. **프로젝트 만들기**를 클릭하고 이름을 입력합니다 (예: "인터뷰_2026_01").
3. 선택적으로 오디오 파일을 추가합니다 (WAV, MP3, M4A, FLAC, OGG, OPUS, MP4, AAC).
4. 생성 후 프로젝트가 활성화됩니다 — 상단 바에서 확인할 수 있습니다.

### 열기 및 관리
- 프로젝트 카드를 클릭하여 열고 활성 프로젝트로 설정합니다.
- 프로젝트를 `.aistate` 파일로 내보내기 (카드 컨텍스트 메뉴) — 다른 기기로 전송할 수 있습니다.
- `.aistate` 파일을 가져와 다른 인스턴스의 프로젝트를 추가합니다.

### 삭제
- 카드 컨텍스트 메뉴에서 프로젝트를 삭제합니다. 파일 덮어쓰기 방법을 선택할 수 있습니다 (빠른 삭제 / 의사난수 / HMG IS5 / Gutmann).

## 2. 전사

음성 인식(Speech-to-Text) 모듈입니다.

### 사용 방법
1. 오디오 파일이 있는 활성 프로젝트가 있는지 확인합니다 (또는 도구 모음 버튼으로 추가).
2. **ASR 엔진** (Whisper 또는 NeMo)과 **모델** (예: `large-v3`)을 선택합니다.
3. 녹음 **언어**를 선택합니다 (자동 감지의 경우 `auto`).
4. **전사** 버튼 (AI 아이콘)을 클릭합니다.

### 결과
- 텍스트가 타임스탬프가 있는 블록으로 표시됩니다 (`[00:00:05.120 - 00:00:08.340]`).
- 블록을 **클릭**하면 해당 오디오 세그먼트를 재생합니다.
- 블록을 **우클릭**하면 인라인 편집기가 열립니다 — 텍스트와 화자 이름을 변경할 수 있습니다.
- 모든 변경사항은 자동으로 저장됩니다.

### 소리 감지
- 소리 감지 모델이 설치된 경우 (YAMNet, PANNs, BEATs), 도구 모음에서 **소리 감지** 옵션을 활성화합니다.
- 감지된 소리 (기침, 웃음, 음악, 사이렌 등)가 텍스트에 마커로 표시됩니다.

### 텍스트 교정
- **교정** 기능을 사용하여 LLM 모델(예: Bielik, PLLuM, Qwen3)로 전사 내용을 자동 교정합니다.
- 원본과 교정된 텍스트를 나란히 비교(diff 뷰)할 수 있습니다.

### 메모
- **메모** 패널 (오른쪽)에서 전체 메모와 개별 블록 메모를 추가할 수 있습니다.
- 각 블록 옆의 메모 아이콘은 메모가 할당되었는지 나타냅니다.

### 보고서
- 도구 모음에서 형식 (HTML, DOC, TXT)을 선택하고 **저장**을 클릭합니다 — 보고서는 프로젝트 폴더에 저장됩니다.

## 3. 화자분리

화자 식별 모듈 — "누가 언제 말하는가".

### 사용 방법
1. 오디오 파일이 있는 활성 프로젝트가 필요합니다.
2. **화자분리 엔진**을 선택합니다: pyannote (오디오) 또는 NeMo 화자분리.
3. 선택적으로 화자 수를 설정합니다 (또는 자동으로 둡니다).
4. **화자분리**를 클릭합니다.

### 결과
- 각 블록에 화자 라벨이 있습니다 (예: `SPEAKER_00`, `SPEAKER_01`).
- **화자 매핑**: 라벨을 이름으로 교체합니다 (예: `SPEAKER_00` → "김철수").
- 필드에 이름을 입력 → **매핑 적용** 클릭 → 라벨이 교체됩니다.
- 매핑은 `project.json`에 저장됩니다 — 프로젝트를 다시 열 때 자동으로 불러옵니다.

### 편집
- 블록을 우클릭하여 편집기를 엽니다: 텍스트 변경, 화자 변경, 세그먼트 재생.
- 메모는 전사와 동일하게 작동합니다.

### 보고서
- 도구 모음에서 HTML / DOC / TXT로 결과를 내보냅니다.

## 4. 번역

NLLB 모델 (Meta) 기반 다국어 번역 모듈입니다.

### 사용 방법
1. **번역** 탭으로 이동합니다.
2. **NLLB 모델**을 선택합니다 (NLLB 설정에서 설치되어 있어야 합니다).
3. 텍스트를 붙여넣거나 문서를 가져옵니다 (TXT, DOCX, PDF, SRT).
4. **원본 언어**와 **대상 언어**를 선택합니다 (여러 개 선택 가능).
5. **생성**을 클릭합니다.

### 모드
- **빠른 (NLLB)** — 소형 모델, 빠른 번역.
- **정확 (NLLB)** — 대형 모델, 높은 품질.

### 추가 기능
- **서식 유지** — 단락과 줄바꿈을 유지합니다.
- **용어 사전** — 전문 용어 사전을 사용합니다.
- **TTS (리더)** — 원문과 번역을 듣습니다 (TTS 엔진 설치 필요).
- **프리셋** — 미리 만들어진 설정 (비즈니스 문서, 학술 논문, 오디오 전사).

### 보고서
- 결과를 HTML / DOC / TXT로 내보냅니다.

## 5. Chat LLM

로컬 LLM 모델과의 채팅 인터페이스입니다 (Ollama 경유).

### 사용 방법
1. **Chat LLM**으로 이동합니다.
2. 목록에서 **모델**을 선택합니다 (Ollama에 설치되어 있어야 합니다).
3. 메시지를 입력하고 **보내기**를 클릭합니다.

### 옵션
- **시스템 프롬프트** — 어시스턴트의 역할을 정의합니다 (예: "당신은 한국법 전문 변호사입니다").
- **온도** — 응답의 창의성을 제어합니다 (0 = 결정적, 1.5 = 매우 창의적).
- **기록** — 대화가 자동으로 저장됩니다. 사이드바에서 이전 대화로 돌아갈 수 있습니다.

## 6. 분석

분석 탭에는 LLM, AML, GSM, Crypto 네 가지 모듈이 있습니다. 상단 탭으로 전환합니다.

### 6.1 LLM 분석

LLM 모델을 사용한 콘텐츠 분석 모듈입니다.

1. 사이드바 패널에서 **데이터 소스**를 선택합니다 (전사, 화자분리, 메모, 문서).
2. **프롬프트**를 선택합니다 — 템플릿 또는 직접 생성.
3. **생성** (도구 모음의 AI 아이콘)을 클릭합니다.

#### 빠른 분석
- 전사 후 자동으로 실행되는 경량 분석.
- 소형 모델 사용 (LLM 설정에서 구성).

#### 심층 분석
- 선택한 소스와 프롬프트에 기반한 전체 분석.
- 사용자 정의 프롬프트 지원: "사용자 정의 프롬프트" 필드에 지시를 입력합니다 (예: "결정사항이 포함된 회의록 작성").

### 6.2 AML 분석 (자금세탁방지)

은행 명세서 금융 분석 모듈입니다.

1. 은행 명세서 (PDF 또는 MT940) 업로드 — 시스템이 자동으로 은행을 감지하고 거래를 파싱합니다.
2. **명세서 정보**, 식별된 계좌 및 카드를 검토합니다.
3. 거래를 분류합니다: 중립 / 합법 / 의심 / 모니터링.
4. **차트** 확인: 시간별 잔액, 카테고리, 채널, 월별 추세, 일별 활동, 주요 거래 상대방.
5. **ML 이상 감지** — Isolation Forest 알고리즘이 비정상적인 거래를 감지합니다.
6. **플로우 그래프** — 거래 상대방 관계 시각화 (레이아웃: 흐름, 금액, 시간순).
7. LLM 모델에 금융 데이터에 대한 질문 ("분석을 위한 질문 / 지시" 섹션).
8. 분석 결과가 포함된 **HTML 보고서** 다운로드.

#### 분석 패널 (AML)
- 검색, 전체 메모, 요소별 메모가 있는 왼쪽 패널.
- **Ctrl+M** — 현재 요소에 빠르게 메모 추가.
- 태그: 중립, 합법, 의심, 모니터링 + 사용자 정의 태그 4개 (더블 클릭으로 이름 변경).

### 6.3 GSM / BTS 분석

GSM 빌링 데이터 분석 모듈입니다.

1. 빌링 데이터를 불러옵니다 (CSV, XLSX, PDF, 다중 파일 ZIP).
2. **요약** 확인: 레코드 수, 기간, 기기 (IMEI/IMSI).
3. **이상 감지** — 비정상적인 패턴 감지 (야간 활동, 로밍, 듀얼 SIM 등).
4. **특수 번호** — 긴급, 서비스 번호 등 식별.
5. **연락처 그래프** — 가장 빈번한 연락처 시각화 (Top 5/10/15/20).
6. **레코드** — 필터링, 검색, 열 관리가 가능한 전체 레코드 테이블.
7. **활동 차트** — 시간대별 분포 히트맵, 야간 및 주말 활동.
8. **BTS 지도** — 다양한 뷰가 포함된 인터랙티브 지도:
   - 전체 포인트, 경로, 클러스터, 이동, 국경, BTS 커버리지, 히트맵, 타임라인.
   - **오버레이**: 군사 기지, 민간 공항, 외교 공관.
   - **KML/KMZ 가져오기** — Google Earth의 사용자 정의 레이어.
   - **오프라인 지도** — MBTiles 지원 (래스터 + 벡터 PBF).
   - **영역 선택** — 공간 쿼리를 위한 원형 / 사각형.
9. **감지된 위치** — 가장 빈번한 위치의 클러스터.
10. **국경 통과** — 해외 출국 감지.
11. **외박** — 숙박 장소 분석.
12. **내러티브 분석 (LLM)** — Ollama 모델을 사용하여 GSM 분석 보고서 생성.
13. **보고서** — HTML / DOCX / TXT로 내보내기. 차트가 포함된 분석 노트 DOCX.

#### 섹션 레이아웃
- 분석 패널의 **레이아웃 맞춤** 버튼으로 섹션의 순서와 표시 여부를 변경합니다 (드래그 / 체크박스).

#### 분석 패널 (GSM)
- 검색, 전체 메모, 요소별 메모가 있는 왼쪽 패널.
- **Ctrl+M** — 현재 레코드에 빠르게 메모 추가.

#### 독립형 지도
- 빌링 데이터 없이 지도를 엽니다 (도구 모음의 지도 버튼).
- 편집 모드 — 포인트, 폴리곤, 사용자 레이어 추가.

### 6.4 암호화폐 분석

암호화폐 거래 분석 모듈입니다.

1. 거래 데이터를 불러옵니다 (CSV, JSON).
2. 분석 결과를 검토합니다.
3. LLM 모델을 사용하여 내러티브 분석을 생성합니다.

## 7. 로그

작업 모니터링 및 시스템 진단입니다.

- **로그** 탭으로 이동하여 모든 작업의 상태를 확인합니다 (전사, 화자분리, 분석, 번역).
- 로그를 클립보드에 복사하거나 파일로 저장합니다.
- 작업 목록을 지웁니다 (프로젝트는 삭제되지 않습니다).

## 8. 관리자 패널

### GPU 설정
- GPU 카드, VRAM, 활성 작업을 모니터링합니다.
- 동시성 제한을 설정합니다 (GPU당 슬롯, 메모리 비율).
- 작업 큐를 확인하고 관리합니다.
- 작업 유형 우선순위를 설정합니다 (드래그하여 순서 변경).

### ASR 설정
- Whisper 모델을 설치합니다 (tiny → large-v3).
- NeMo ASR 및 화자분리 모델을 설치합니다.
- 소리 감지 모델을 설치합니다 (YAMNet, PANNs, BEATs).

### LLM 설정
- Ollama 모델을 찾아보고 설치합니다 (빠른 분석, 심층 분석, 금융, 교정, 번역, 비전/OCR).
- 사용자 정의 Ollama 모델을 추가합니다.
- 토큰을 설정합니다 (Hugging Face).

### NLLB 설정
- NLLB 번역 모델을 설치합니다 (distilled-600M, distilled-1.3B, base-3.3B).
- 모델 정보를 확인합니다 (크기, 품질, 요구 사항).

### TTS 설정
- 리더 엔진을 설치합니다: Piper (빠름, CPU), MMS (1100개 이상의 언어), Kokoro (최고 품질).
- 사용 전 음성을 테스트합니다.

## 9. 설정

- **UI 언어** — PL / EN / KO 간 전환.
- **Hugging Face 토큰** — pyannote 모델에 필요합니다 (게이트된 모델).

## 10. 사용자 관리 (멀티 유저 모드)

멀티 유저 모드가 활성화된 경우:
- 관리자가 사용자 계정을 생성, 편집, 차단 및 삭제합니다.
- 새 사용자는 가입 후 관리자 승인을 기다립니다.
- 각 사용자에게는 사용 가능한 모듈을 결정하는 역할이 할당됩니다.
