# AISTATE Web — 사용자 매뉴얼

---

## 1. 프로젝트

프로젝트는 AISTATE 작업의 핵심 요소입니다. 각 프로젝트에는 오디오 파일, 전사 결과, 화자분리, 번역, 분석 및 메모가 저장됩니다.

### 프로젝트 만들기
1. 사이드바에서 **프로젝트** 탭으로 이동합니다.
2. **프로젝트 만들기**를 클릭하고 이름을 입력합니다 (예: "인터뷰_2026_01").
3. 선택적으로 오디오 파일을 추가합니다 (WAV, MP3, M4A, FLAC, OGG, OPUS, MP4, AAC).
4. 생성 후 프로젝트가 활성화됩니다 — 상단 바에서 확인할 수 있습니다.

### 열기 및 관리
- 프로젝트 카드를 클릭하여 열고 활성 프로젝트로 설정합니다.
- 프로젝트를 `.aistate` 파일로 내보내기 (카드 컨텍스트 메뉴) — 다른 기기로 전송할 수 있습니다.
- `.aistate` 파일을 가져와 다른 인스턴스의 프로젝트를 추가합니다.

### 삭제
- 카드 컨텍스트 메뉴에서 프로젝트를 삭제합니다. 파일 덮어쓰기 방법을 선택할 수 있습니다 (빠른 삭제 / 의사난수 / HMG IS5 / Gutmann).

## 2. 전사

음성 인식(Speech-to-Text) 모듈입니다.

### 사용 방법
1. 오디오 파일이 있는 활성 프로젝트가 있는지 확인합니다 (또는 도구 모음 버튼으로 추가).
2. **ASR 엔진** (Whisper 또는 NeMo)과 **모델** (예: `large-v3`)을 선택합니다.
3. 녹음 **언어**를 선택합니다 (자동 감지의 경우 `auto`).
4. **전사** 버튼 (AI 아이콘)을 클릭합니다.

### 결과
- 텍스트가 타임스탬프가 있는 블록으로 표시됩니다 (`[00:00:05.120 - 00:00:08.340]`).
- 블록을 **클릭**하면 해당 오디오 세그먼트를 재생합니다.
- 블록을 **우클릭**하면 인라인 편집기가 열립니다 — 텍스트와 화자 이름을 변경할 수 있습니다.
- 모든 변경사항은 자동으로 저장됩니다.

### 소리 감지
- 소리 감지 모델이 설치된 경우 (YAMNet, PANNs, BEATs), 도구 모음에서 **소리 감지** 옵션을 활성화합니다.
- 감지된 소리 (기침, 웃음, 음악, 사이렌 등)가 텍스트에 마커로 표시됩니다.

### 텍스트 교정
- **교정** 기능을 사용하여 LLM 모델(예: Bielik, PLLuM, Qwen3)로 전사 내용을 자동 교정합니다.
- 원본과 교정된 텍스트를 나란히 비교(diff 뷰)할 수 있습니다.

### 메모
- **메모** 패널 (오른쪽)에서 전체 메모와 개별 블록 메모를 추가할 수 있습니다.
- 각 블록 옆의 메모 아이콘은 메모가 할당되었는지 나타냅니다.

### 보고서
- 도구 모음에서 형식 (HTML, DOC, TXT)을 선택하고 **저장**을 클릭합니다 — 보고서는 프로젝트 폴더에 저장됩니다.

## 3. 화자분리

화자 식별 모듈 — "누가 언제 말하는가".

### 사용 방법
1. 오디오 파일이 있는 활성 프로젝트가 필요합니다.
2. **화자분리 엔진**을 선택합니다: pyannote (오디오) 또는 NeMo 화자분리.
3. 선택적으로 화자 수를 설정합니다 (또는 자동으로 둡니다).
4. **화자분리**를 클릭합니다.

### 결과
- 각 블록에 화자 라벨이 있습니다 (예: `SPEAKER_00`, `SPEAKER_01`).
- **화자 매핑**: 라벨을 이름으로 교체합니다 (예: `SPEAKER_00` → "김철수").
- 필드에 이름을 입력 → **매핑 적용** 클릭 → 라벨이 교체됩니다.
- 매핑은 `project.json`에 저장됩니다 — 프로젝트를 다시 열 때 자동으로 불러옵니다.

### 편집
- 블록을 우클릭하여 편집기를 엽니다: 텍스트 변경, 화자 변경, 세그먼트 재생.
- 메모는 전사와 동일하게 작동합니다.

### 보고서
- 도구 모음에서 HTML / DOC / TXT로 결과를 내보냅니다.

## 4. 번역

NLLB 모델 (Meta) 기반 다국어 번역 모듈입니다.

### 사용 방법
1. **번역** 탭으로 이동합니다.
2. **NLLB 모델**을 선택합니다 (NLLB 설정에서 설치되어 있어야 합니다).
3. 텍스트를 붙여넣거나 문서를 가져옵니다 (TXT, DOCX, PDF, SRT).
4. **원본 언어**와 **대상 언어**를 선택합니다 (여러 개 선택 가능).
5. **생성**을 클릭합니다.

### 모드
- **빠른 (NLLB)** — 소형 모델, 빠른 번역.
- **정확 (NLLB)** — 대형 모델, 높은 품질.

### 추가 기능
- **서식 유지** — 단락과 줄바꿈을 유지합니다.
- **용어 사전** — 전문 용어 사전을 사용합니다.
- **TTS (리더)** — 원문과 번역을 듣습니다 (TTS 엔진 설치 필요).
- **프리셋** — 미리 만들어진 설정 (비즈니스 문서, 학술 논문, 오디오 전사).

### 보고서
- 결과를 HTML / DOC / TXT로 내보냅니다.

## 5. Chat LLM

로컬 LLM 모델과의 채팅 인터페이스입니다 (Ollama 경유).

### 사용 방법
1. **Chat LLM**으로 이동합니다.
2. 목록에서 **모델**을 선택합니다 (Ollama에 설치되어 있어야 합니다).
3. 메시지를 입력하고 **보내기**를 클릭합니다.

### 옵션
- **시스템 프롬프트** — 어시스턴트의 역할을 정의합니다 (예: "당신은 한국법 전문 변호사입니다").
- **온도** — 응답의 창의성을 제어합니다 (0 = 결정적, 1.5 = 매우 창의적).
- **기록** — 대화가 자동으로 저장됩니다. 사이드바에서 이전 대화로 돌아갈 수 있습니다.

## 6. 분석

분석 탭에는 LLM, AML, GSM, Crypto 네 가지 모듈이 있습니다. 상단 탭으로 전환합니다.

### 6.1 LLM 분석

LLM 모델을 사용한 콘텐츠 분석 모듈입니다.

1. 사이드바 패널에서 **데이터 소스**를 선택합니다 (전사, 화자분리, 메모, 문서).
2. **프롬프트**를 선택합니다 — 템플릿 또는 직접 생성.
3. **생성** (도구 모음의 AI 아이콘)을 클릭합니다.

#### 빠른 분석
- 전사 후 자동으로 실행되는 경량 분석.
- 소형 모델 사용 (LLM 설정에서 구성).

#### 심층 분석
- 선택한 소스와 프롬프트에 기반한 전체 분석.
- 사용자 정의 프롬프트 지원: "사용자 정의 프롬프트" 필드에 지시를 입력합니다 (예: "결정사항이 포함된 회의록 작성").

### 6.2 AML 분석 (자금세탁방지)

은행 명세서 금융 분석 모듈입니다.

1. 은행 명세서 (PDF 또는 MT940) 업로드 — 시스템이 자동으로 은행을 감지하고 거래를 파싱합니다.
2. **명세서 정보**, 식별된 계좌 및 카드를 검토합니다.
3. 거래를 분류합니다: 중립 / 합법 / 의심 / 모니터링.
4. **차트** 확인: 시간별 잔액, 카테고리, 채널, 월별 추세, 일별 활동, 주요 거래 상대방.
5. **ML 이상 감지** — Isolation Forest 알고리즘이 비정상적인 거래를 감지합니다.
6. **플로우 그래프** — 거래 상대방 관계 시각화 (레이아웃: 흐름, 금액, 시간순).
7. LLM 모델에 금융 데이터에 대한 질문 ("분석을 위한 질문 / 지시" 섹션).
8. 분석 결과가 포함된 **HTML 보고서** 다운로드.

#### 분석 패널 (AML)
- 검색, 전체 메모, 요소별 메모가 있는 왼쪽 패널.
- **Ctrl+M** — 현재 요소에 빠르게 메모 추가.
- 태그: 중립, 합법, 의심, 모니터링 + 사용자 정의 태그 4개 (더블 클릭으로 이름 변경).

### 6.3 GSM / BTS 분석

GSM 빌링 데이터 분석 모듈입니다.

1. 빌링 데이터를 불러옵니다 (CSV, XLSX, PDF, 다중 파일 ZIP).
2. **요약** 확인: 레코드 수, 기간, 기기 (IMEI/IMSI).
3. **이상 감지** — 비정상적인 패턴 감지 (야간 활동, 로밍, 듀얼 SIM 등).
4. **특수 번호** — 긴급, 서비스 번호 등 식별.
5. **연락처 그래프** — 가장 빈번한 연락처 시각화 (Top 5/10/15/20).
6. **레코드** — 필터링, 검색, 열 관리가 가능한 전체 레코드 테이블.
7. **활동 차트** — 시간대별 분포 히트맵, 야간 및 주말 활동.
8. **BTS 지도** — 다양한 뷰가 포함된 인터랙티브 지도:
   - 전체 포인트, 경로, 클러스터, 이동, 국경, BTS 커버리지, 히트맵, 타임라인.
   - **오버레이**: 군사 기지, 민간 공항, 외교 공관.
   - **KML/KMZ 가져오기** — Google Earth의 사용자 정의 레이어.
   - **오프라인 지도** — MBTiles 지원 (래스터 + 벡터 PBF).
   - **영역 선택** — 공간 쿼리를 위한 원형 / 사각형.
9. **감지된 위치** — 가장 빈번한 위치의 클러스터.
10. **국경 통과** — 해외 출국 감지.
11. **외박** — 숙박 장소 분석.
12. **내러티브 분석 (LLM)** — Ollama 모델을 사용하여 GSM 분석 보고서 생성.
13. **보고서** — HTML / DOCX / TXT로 내보내기. 차트가 포함된 분석 노트 DOCX.

#### 섹션 레이아웃
- 분석 패널의 **레이아웃 맞춤** 버튼으로 섹션의 순서와 표시 여부를 변경합니다 (드래그 / 체크박스).

#### 분석 패널 (GSM)
- 검색, 전체 메모, 요소별 메모가 있는 왼쪽 패널.
- **Ctrl+M** — 현재 레코드에 빠르게 메모 추가.

#### 독립형 지도
- 빌링 데이터 없이 지도를 엽니다 (도구 모음의 지도 버튼).
- 편집 모드 — 포인트, 폴리곤, 사용자 레이어 추가.

### 6.4 암호화폐 분석 *(실험적)*

오프라인 암호화폐 거래 분석 모듈 (BTC / ETH) 및 거래소 데이터입니다.

#### 데이터 가져오기
1. 분석 모듈의 **Crypto** 탭으로 이동합니다.
2. **데이터 불러오기**를 클릭하고 CSV 또는 JSON 파일을 선택합니다.
3. 시스템이 자동으로 형식을 감지합니다:
   - **블록체인**: WalletExplorer.com, Etherscan
   - **거래소**: Binance, Kraken, Coinbase, Revolut 등 (16개 이상 형식)
4. 불러오기 후 데이터 정보가 표시됩니다: 거래 수, 기간, 토큰 포트폴리오.

#### 데이터 보기
- **거래소 모드** — 유형별 거래소 거래 테이블 (입금, 출금, 스왑, 스테이킹 등).
- **블록체인 모드** — 주소와 금액이 포함된 온체인 거래 테이블.
- **토큰 포트폴리오** — 설명, 분류 (알려진 / 미확인), 가치가 포함된 토큰 목록.
- **거래 유형 사전** — 유형 위에 마우스를 올리면 설명이 표시됩니다 (툴팁).

#### 분류 및 검토
- 거래를 분류합니다: 중립 / 합법 / 의심 / 모니터링.
- 시스템이 패턴을 기반으로 일부 거래를 자동 분류합니다.

#### 이상 감지
- **ML 이상 감지** — 알고리즘이 비정상적인 거래를 감지합니다 (대량 금액, 비정상 시간, 의심스러운 패턴).
- 이상 유형: peel chain, dust attack, round-trip, smurfing, structuring.
- **OFAC** 제재 주소 데이터베이스 및 알려진 DeFi 컨트랙트 조회.

#### 차트
- **잔액 타임라인** — 시간에 따른 잔액 변화 (로그 정규화 포함).
- **월간 거래량** — 월별 거래 합계.
- **일일 활동** — 요일별 거래 분포.
- **거래 상대방 순위** — 가장 빈번한 거래 파트너.

#### 플로우 그래프
- 인터랙티브 **거래 그래프** (Cytoscape.js) — 주소/거래 상대방 간 흐름 시각화.
- 노드를 클릭하면 세부 정보를 볼 수 있습니다.

#### 사용자 프로파일링 (Binance)
- 10가지 행동 패턴: HODLer, Scalper, Day Trader, Swing Trader, Staker, Whale, Institutional, Alpha Hunter, Meme Trader, Bagholder.
- 18개 포렌식 분석 카드 (내부 거래 상대방, 온체인 주소, wash trading, P2P, 수수료 분석 등).

#### 내러티브 분석 (LLM)
- **분석 생성** 클릭 → Ollama 모델이 결론과 권장 사항이 포함된 서술형 보고서를 생성합니다.

#### 보고서
- 도구 모음에서 **HTML / DOCX / TXT**로 결과를 내보냅니다.

#### 분석 패널 (Crypto)
- 전체 메모와 거래별 메모가 있는 왼쪽 패널.
- **Ctrl+M** — 현재 거래에 빠르게 메모 추가.
- 태그: 중립, 합법, 의심, 모니터링 + 사용자 정의 태그 4개.

## 7. 로그

작업 모니터링 및 시스템 진단입니다.

- **로그** 탭으로 이동하여 모든 작업의 상태를 확인합니다 (전사, 화자분리, 분석, 번역).
- 로그를 클립보드에 복사하거나 파일로 저장합니다.
- 작업 목록을 지웁니다 (프로젝트는 삭제되지 않습니다).

## 8. 관리자 패널

### GPU 설정
- GPU 카드, VRAM, 활성 작업을 모니터링합니다.
- 동시성 제한을 설정합니다 (GPU당 슬롯, 메모리 비율).
- 작업 큐를 확인하고 관리합니다.
- 작업 유형 우선순위를 설정합니다 (드래그하여 순서 변경).

### ASR 설정
- Whisper 모델을 설치합니다 (tiny → large-v3).
- NeMo ASR 및 화자분리 모델을 설치합니다.
- 소리 감지 모델을 설치합니다 (YAMNet, PANNs, BEATs).

### LLM 설정
- Ollama 모델을 찾아보고 설치합니다 (빠른 분석, 심층 분석, 금융, 교정, 번역, 비전/OCR).
- 사용자 정의 Ollama 모델을 추가합니다.
- 토큰을 설정합니다 (Hugging Face).

### NLLB 설정
- NLLB 번역 모델을 설치합니다 (distilled-600M, distilled-1.3B, base-3.3B).
- 모델 정보를 확인합니다 (크기, 품질, 요구 사항).

### TTS 설정
- 리더 엔진을 설치합니다: Piper (빠름, CPU), MMS (1100개 이상의 언어), Kokoro (최고 품질).
- 사용 전 음성을 테스트합니다.

## 11. 프로젝트 암호화

AISTATE는 무단 접근으로부터 데이터를 보호하기 위해 프로젝트 암호화를 지원합니다.

### 설정 (관리자)

**사용자 관리 → 보안 → 보안 정책** 패널에서 관리자가 설정합니다:

- **프로젝트 암호화** — 암호화 기능 활성화 / 비활성화.
- **암호화 방법** — 세 가지 방법 중 하나를 선택합니다:

| 수준 | 알고리즘 | 설명 |
|------|----------|------|
| **경량** | AES-128-GCM | 빠른 암호화, 일반적인 접근으로부터 보호 |
| **표준** | AES-256-GCM | 기본 수준 — 속도와 보안의 균형 |
| **최대** | AES-256-GCM + ChaCha20-Poly1305 | 민감한 데이터를 위한 이중 레이어 암호화 |

- **암호화 강제** — 활성화 시 사용자가 암호화되지 않은 프로젝트를 만들 수 없습니다.

선택한 암호화 수준은 이후 사용자가 만드는 모든 프로젝트에 적용됩니다.

### 암호화된 프로젝트 만들기

프로젝트 생성 시 현재 방법에 대한 정보 (예: "AES-256-GCM")와 함께 **프로젝트 암호화** 체크박스가 나타납니다. 관리자가 암호화를 활성화한 경우 체크박스는 기본적으로 선택되며, 암호화가 강제된 경우에는 잠깁니다.

### 내보내기 및 가져오기

- 암호화된 프로젝트의 **내보내기** — `.aistate` 파일은 항상 암호화됩니다. 시스템이 **내보내기 비밀번호** (계정 비밀번호와 별도)를 요청합니다.
- **가져오기** — 시스템이 `.aistate` 파일의 암호화 여부를 자동으로 감지합니다. 암호화된 경우 — 비밀번호를 요청합니다. 가져온 후 관리자의 현재 정책에 따라 프로젝트가 다시 암호화됩니다.
- 암호화되지 않은 프로젝트는 비밀번호 없이 내보내거나 "내보내기 암호화" 옵션을 사용할 수 있습니다.

### <span style="color:red">⚠ 접근 복구 — 단계별 절차</span>

<span style="color:red">각 암호화된 프로젝트에는 사용자의 키 (비밀번호에서 파생)로 보호되는 무작위 암호화 키 (Project Key)가 있습니다. 또한 프로젝트 키는 관리자의 **마스터 키 (Master Key)**로 보호됩니다. 관리자는 **단독으로 프로젝트를 복호화할 수 없으며** — 사용자와의 상호작용이 필요합니다.</span>

#### <span style="color:red">시나리오 1: 사용자가 비밀번호를 잊은 경우 (자체 복구)</span>

<span style="color:red">사용자가 복구 구문 (계정 생성 시 받은 12개 단어)을 보유하고 있는 경우.</span>

<span style="color:red">**사용자 단계:**</span>
<span style="color:red">1. 로그인 화면에서 **"비밀번호를 잊었습니다"**를 클릭합니다.</span>
<span style="color:red">2. **복구 구문** (12개 단어, 공백으로 구분)을 입력합니다.</span>
<span style="color:red">3. 시스템이 구문을 확인합니다 — 올바르면 새 비밀번호 양식이 나타납니다.</span>
<span style="color:red">4. **새 비밀번호**를 설정하고 확인합니다.</span>
<span style="color:red">5. 시스템이 자동으로 모든 암호화된 프로젝트의 키를 새 비밀번호로 다시 암호화합니다.</span>
<span style="color:red">6. 새 비밀번호로 정상적으로 로그인합니다.</span>

<span style="color:red">**관리자 개입이 필요 없습니다** — 전체 프로세스가 자동입니다.</span>

#### <span style="color:red">시나리오 2: 사용자가 비밀번호를 잊었지만 복구 구문을 가지고 있는 경우 (관리자 지원 복구)</span>

<span style="color:red">자체 비밀번호 재설정이 작동하지 않거나 정책에 의해 비활성화된 경우:</span>

<span style="color:red">**관리자 단계:**</span>
<span style="color:red">1. **사용자 관리** → 사용자 계정을 찾습니다.</span>
<span style="color:red">2. **"복구 토큰 생성"**을 클릭합니다 — 시스템이 일회용 토큰을 생성합니다 (24시간 유효).</span>
<span style="color:red">3. 토큰을 사용자에게 전달합니다 (직접, 전화 또는 다른 보안 채널을 통해).</span>

<span style="color:red">**사용자 단계:**</span>
<span style="color:red">1. **접근 복구** 페이지로 이동합니다 (로그인 화면의 링크).</span>
<span style="color:red">2. 관리자로부터 받은 **복구 토큰**을 입력합니다.</span>
<span style="color:red">3. **복구 구문** (12개 단어)을 입력합니다.</span>
<span style="color:red">4. **새 비밀번호**를 설정합니다.</span>
<span style="color:red">5. 시스템이 새 비밀번호로 프로젝트 키를 다시 암호화합니다.</span>
<span style="color:red">6. 토큰은 사용 후 무효화됩니다.</span>

#### <span style="color:red">시나리오 3: 사용자가 비밀번호와 복구 구문을 모두 잃은 경우 (마스터 키 복구)</span>

<span style="color:red">**마스터 키**가 사용되는 유일한 시나리오입니다.</span>

<span style="color:red">**관리자 단계:**</span>
<span style="color:red">1. **사용자 관리 → 보안 → 암호화**를 엽니다.</span>
<span style="color:red">2. 마스터 키 잠금 해제를 위해 **관리자 비밀번호**를 입력합니다.</span>
<span style="color:red">3. 접근 권한을 잃은 사용자 계정을 선택합니다.</span>
<span style="color:red">4. **"긴급 복구"**를 클릭합니다 — 시스템이 마스터 키를 사용하여 사용자의 프로젝트 키를 복호화합니다.</span>
<span style="color:red">5. 시스템이 사용자를 위한 **새 복구 구문**을 생성합니다.</span>
<span style="color:red">6. 시스템이 **일회용 복구 토큰**을 생성합니다.</span>
<span style="color:red">7. 사용자에게 전달합니다: 토큰 + 새 복구 구문.</span>

<span style="color:red">**사용자 단계:**</span>
<span style="color:red">1. **접근 복구** 페이지로 이동합니다.</span>
<span style="color:red">2. 관리자로부터 받은 **토큰**을 입력합니다.</span>
<span style="color:red">3. 관리자로부터 받은 **새 복구 구문**을 입력합니다.</span>
<span style="color:red">4. **새 비밀번호**를 설정합니다.</span>
<span style="color:red">5. 시스템이 새 비밀번호로 프로젝트 키를 다시 암호화합니다.</span>

<span style="color:red">**중요:** 새 복구 구문을 즉시 기록하고 안전한 장소에 보관해야 합니다!</span>

### <span style="color:red">⚠ 마스터 키 백업</span>

<span style="color:red">**경고:** 사용자가 비밀번호와 복구 구문을 잃고, 관리자가 마스터 키를 분실하면 — **암호화된 프로젝트의 데이터는 복구가 불가능합니다**. "백도어"는 없습니다.</span>

<span style="color:red">**관리자 책임:**</span>
<span style="color:red">1. 마스터 키 초기화 후 암호화 패널에서 **"마스터 키 백업"**을 클릭합니다.</span>
<span style="color:red">2. 관리자 비밀번호를 입력합니다 — 시스템이 base64 형식으로 키를 표시합니다.</span>
<span style="color:red">3. **오프라인 매체에 키를 저장합니다** (USB 드라이브, 금고 속 인쇄물) — 시스템이나 이메일에 저장하지 마십시오.</span>
<span style="color:red">4. **"마스터 키 확인"** 버튼을 사용하여 정기적으로 백업을 검증합니다.</span>

<span style="color:red">**마스터 키 분실 + 사용자 비밀번호 분실 + 복구 구문 분실 = 영구적인 데이터 손실.**</span>

### <span style="color:red">⚠ 암호화된 프로젝트에서의 검색</span>

<span style="color:red">프로젝트 목록 (이름, 생성 날짜)은 항상 표시됩니다. 그러나 **콘텐츠 검색** (전사, 메모, 분석 결과)은 데이터 복호화가 필요하며 **활성화된 (열린) 프로젝트에서만** 작동합니다. 여러 암호화된 프로젝트를 동시에 검색하는 것은 불가능합니다.</span>

## 12. A.R.I.A. — AI 어시스턴트

화면 오른쪽 하단의 A.R.I.A. 플로팅 버튼을 누르면 AI 어시스턴트 패널이 열립니다.

### 기능
- **AI 채팅** — 현재 컨텍스트 (전사, 분석, 데이터)에 대해 질문합니다.
- **자동 컨텍스트** — 어시스턴트가 현재 열린 페이지의 데이터를 자동으로 포함합니다.
- **응답 읽기** (TTS) — 어시스턴트의 응답을 음성으로 듣습니다.
- **힌트 칩** — 현재 모듈에 맞는 미리 만들어진 질문들입니다.
- **드래그 가능** — A.R.I.A. 버튼을 화면 어디든 끌어다 놓을 수 있습니다 (위치가 기억됩니다).

## 13. 오디오 플레이어

프로젝트에 오디오 파일이 있을 때 전사 및 화자분리에서 오디오 플레이어 바가 나타납니다.

- **재생 / 일시정지** — 녹음을 재생하거나 멈춥니다.
- **건너뛰기** ±5초 (버튼 또는 진행 바 클릭).
- **재생 속도** — 0.5×, 0.75×, 1×, 1.25×, 1.5×, 2× (브라우저에 저장).
- **텍스트 세그먼트를 클릭**하면 해당 오디오 구간을 재생합니다.
- **파형 맵** — 세그먼트 마커가 있는 진폭 시각화.

## 14. 검색 및 세그먼트 편집

### 텍스트 검색
- 전사 및 화자분리에서 **Ctrl+F** 또는 도구 모음의 돋보기 아이콘을 사용합니다.
- 검색은 일치 항목을 강조 표시하고 개수를 표시합니다.
- ↑ ↓ 화살표로 일치 항목 간 이동합니다.

### 세그먼트 병합 및 분할
- **세그먼트 병합** — 인접한 두 블록을 선택하고 "병합" (도구 모음 아이콘)을 클릭합니다.
- **세그먼트 분할** — 블록에 커서를 놓고 "분할"을 클릭합니다 → 커서 위치에서 블록이 분할됩니다.
